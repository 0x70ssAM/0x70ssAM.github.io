So I built the map I wish I had when I started.

## What's inside

**17 phases · 5 lifecycle bands · one coherent journey:**

- **Foundation (1–3)** — OS, networking, scripting, governance, threat modeling
- **Develop (4–6)** — Secure coding, SAST, SCA/SBOM, secrets management
- **Build & Deploy (7–9)** — Pipeline hardening, IaC, container & image security
- **Test & Operate (10–12)** — DAST/API security, K8s hardening, runtime/SIEM
- **Govern & Evolve (13–17)** — Vuln management, Zero Trust, supply chain (SLSA), AI/ML security, compliance-as-code

## Built for 2026, not 2020

- AI-driven SAST remediation (Corgea, QwietAI)
- Software supply chain — SBOM + SLSA L3 + Sigstore/Cosign
- eBPF runtime security — Falco + Tetragon
- Zero Trust workload identity — SPIFFE/SPIRE + mTLS
- GitOps delivery — ArgoCD + Kyverno admission gates
- Agentic AI / LLM red-teaming (Garak, LLM Guard, Rebuff)
- Post-quantum crypto readiness + Backstage IDP

Each phase maps to recognized standards — **NIST SSDF, OWASP SAMM/ASVS, SLSA, MITRE ATLAS, CIS** — so you're not just collecting tools, you're building defensible architecture.

## Who this is for

- Engineers transitioning into DevSecOps
- Security teams modernizing legacy AppSec programs
- Platform teams designing secure SDLCs from scratch
- Anyone preparing for senior DevSecOps / Cloud Security roles
